Self Onboard

Self Onboarding API by an iF User for developer program. This API call must contain an OAuth2 bearer token from an iF User that can identify the user account that is requesting onbaording with developer program.

/register put

put

Self onboarding request by an Integratingfactor.com user for developer program. There is no request body.

put /register

Self onboarding request by an Integratingfactor.com user for developer program. There is no request body.

  • Request
  • Response

Headers

  • Authorization: (string)

    Authorization header with OAuth2 bearer token from iF IDP service

    Example:

    Bearer 0cb3887a-7f68-490a-bf15-c89aff96a6fb

HTTP status code 200

onborading success

Body

Type: application/json

Schema:

{
    "title": "Self Onboarding Registration Schema",
    "type": "object",
    "properties": {
        "account_id": {
            "type": "string"
        },
        "user_roles": {
            "type": "array",
            "description": "roles assigned to user for iF Studio org",
            "items": {
                "type" : "string"
            }
        }
    }
}

Example:

{
    "account_id": "3eb64sef-13d3-481d-99z2-14ddf0d982db",
    "user_roles": [
        "USER"
    ]
}

HTTP status code 401

Bearer token authentication failed

Body

Type: application/json

Example:

{
    "error": "user not authenticated"
}

User Onboard

User onboarding APIs for onboarding other iF Users based on their account. This API call must contain an OAuth2 bearer token from an iF User with "ADMIN" role for the project (assigned automatically when user created a project, or when an existing admin explicitly invites/adds another iF User as admin).

/register/{orgId}/accounts get

get

Query all registered Integratingfactor.com users in a project with specified role.

get /register/{orgId}/accounts

Query all registered Integratingfactor.com users in a project with specified role.

  • Request
  • Response

URI Parameters

  • orgId: required (string)

    UUID of project

Headers

  • Authorization: (string)

    Authorization header with OAuth2 bearer token from iF IDP service

    Example:

    Bearer 0cb3887a-7f68-490a-bf15-c89aff96a6fb

Query Parameters

  • role: (string)

    role type to query, should be one of USER | ADMIN | project defined custom role

    Example:

    ?role=USER

HTTP status code 200

found users with specified role

Body

Type: application/json

Schema:

{
    "title": "Get Projects Response Schema",
    "type" : "array",
    "items" : {
        "title": "User Roles Schema",
        "type": "object",
        "properties": {
            "account_id": {
                "type": "string"
            },
            "user_roles": {
                "type": "array",
                "description": "roles assigned to user for the org: cab be USER/ADMIN or any of the custom roles defined for project",
                "items": {
                    "type" : "string"
                }
            }
        }
    }
}

Example:

[
    {
        "account_id": "119edc86-3d49-4436-80bc-0200065007f0",
        "user_roles": [
            "ADMIN", "USER"
        ]
    },
    {
        "account_id": "df3edc86-3d49-4436-80bc-0204566500763",
        "user_roles": [
            "USER"
        ]
    }
]

HTTP status code 401

Bearer token authentication failed

Body

Type: application/json

Example:

{
    "error": "user not authenticated"
}

HTTP status code 403

User is not an ADMIN for the project

Body

Type: application/json

Example:

{
    "error": "you are not a member of this org"
}

HTTP status code 404

no registered project or account

User Invitations

User invitation APIs for inviting other users based on their public identities (e.g. email). This API call must contain an OAuth2 bearer token from an iF User with "ADMIN" role for the project (assigned automatically when user created a project, or when an existing admin explicitly invites/adds another iF User as admin).

If there is already an iF User account with specified user identity, then account is onboarded with provided roles immediately and no pending invitation is kept.

If there is no iF User account with specified identity, then invitation is kept around as pending invitation until an iF User account is created with the specified identity.

/invites/{orgId} get

get

Query all pending invitations in a project with specified role.

get /invites/{orgId}

Query all pending invitations in a project with specified role.

  • Request
  • Response

URI Parameters

  • orgId: required (string)

    UUID of project

Headers

  • Authorization: (string)

    Authorization header with OAuth2 bearer token from iF IDP service

    Example:

    Bearer 0cb3887a-7f68-490a-bf15-c89aff96a6fb

Query Parameters

  • role: (string)

    role type to query, should be one of USER | ADMIN | project defined custom role

    Example:

    ?role=USER

HTTP status code 200

found pending invitations with specified role

Body

Type: application/json

Schema:

{
    "title": "Project Invitation Roles Schema",
    "type" : "array",
    "items" : {
        "title": "Invitation Roles Schema",
        "type": "object",
        "properties": {
            "id_type": {
                "description": "Identity type, must be username | email",
                "type": "string"
            },
            "id_key": {
                "description": "actual identity key of the user",
                "type": "string"
            },
            "user_roles": {
                "type": "array",
                "description": "roles assigned to user for the org: cab be USER/ADMIN or any of the custom roles defined for project",
                "items": {
                    "type" : "string"
                }
            }
        }
    }
}

Example:

[
  {
  "id_key" : "test@gmail.com",
  "id_type" : "email",
  "user_roles" : [
    "USER"
    ]
  },
  {
  "id_key" : "admin@gmail.com",
  "id_type" : "email",
  "user_roles" : [
    "USER",
    "ADMIN"
    ]
  }
]

Example:

}

User is not an ADMIN for the project
